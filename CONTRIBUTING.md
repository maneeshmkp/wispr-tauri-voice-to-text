# Contributing to Wispr

We welcome contributions! Here are some guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add feature: description"`
7. Push to your fork: `git push origin feature/your-feature`
8. Create a Pull Request

## Code Style

- Use consistent indentation (2 spaces for JS, 4 spaces for Rust)
- Write meaningful variable names
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting:
- Test the app in development mode
- Test microphone permissions on your OS
- Test with different audio lengths
- Check error handling

## Reporting Issues

Include:
- OS and version
- Node.js version
- Error message or screenshot
- Steps to reproduce
- Expected vs actual behavior

## Feature Requests

Describe:
- The feature and why it's needed
- Proposed implementation approach
- Any potential challenges

Thank you for contributing to Wispr!
